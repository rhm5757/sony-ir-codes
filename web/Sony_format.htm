Sony IR Code Format

 

Sony’s IR codes are pretty simple, when compared to some other types of IR signals used by other manufacturers.  Sony’s codes have two distinctions: they vary the ON time instead of the off time, and they have three different lengths.  Other than that, it is pretty simple.  However, you need to know a little about IR codes in general.

 

IR stands for Infra-Red, meaning that it is below red in the spectrum.  We cannot see it, of course, however, camcorders can, and if you point any remote control at a camcorder, you should see the light flash.  If you also get it near the antenna of any AM radio, you will also hear beeping noise on all the AM stations.   (Both are good ways to tell if a remote is working.)  IR is used in most consumer electronics because it is very cheap.  The emitters and receivers are both cheap, and you can buy them from Radio Shack or any other electronic parts retailer.  You can think of the emitter as a normal LED, because that is what it is except it emits light we can’t see.  IR signals act like light, that’s why we need to point them at the thing we want to control.

 

The messages contained in IR signals are binary codes, just like everything else dealing with computers.  Most types of code formats, or protocols as they are usually called, transmit some sort of device code and command code.  The device code only varies between different types of electronics, maybe a TV would have the code 1 and a VCR would have 2, and maybe a DVD player would use 3, and so on.  The command code represents the function of the button you pressed, such as “Power”, “1”, “Channel Up”, or anything.  Each command is assigned a number, which stays consistent for the device code.  It may or may not be consistent over all device codes.  For example, in Sony codes, 21 is always the command code for “Power”, but the various arrow keys have different command codes depending on the device.

 

IR signals are usually transmitted by varying the time the IR “light” is off, or on in Sony’s case, or sometimes both the on and off times, this is called pulse width modulation.  Some other protocols like RC5 use bi-phase, which means the bits are off-on and on-off and all the bits are the same length.  A few protocols just turn the light on for a “1” bit and off for a “0” bit, and at least one protocol I’ve seen uses that but reverses the bit’s meanings.  Many IR protocols send the least significant bit first, meaning the first bit is worth 1, the second 2, the third 4, and so on.  This is backwards from what we normally see with numbers in either binary or decimal, we usually put numbers most significant bit first, meaning the last bit is always worth 1.  Many protocols have some redundancy or error-checking mechanism built in.

 

The other important thing to know about IR signals is that when the light is on, it is usually actually flashing rapidly at what is called the carrier frequency, instead of just staying solidly on.  This is done so that the signal can be identified better in a noisy environment.  There are many sources of IR other than remotes, mainly the sun, but also certain light bulbs, and anything that gives off heat also gives off IR.  Usually this frequency is between 25-57 kHz.  Sony uses 40 kHz.  Some protocols, especially ones used by cable boxes, don’t have a carrier frequency and usually send really short bursts.

 

Now for the Sony specifics.  Sony uses a carrier frequency of 40 kHz.  They turn the light on and off in units of 600 microseconds (us).  If you calculate the length of one burst of carrier, you can see that there are exactly 24 bursts of carrier in 600 us.  Sony uses 4 units on and 1 unit off as the first burst, to indicate the beginning of the signal, usually called the lead-in.  This is 2400 us on and 600 us off.  From now on the on and off times will be represented by +x -x, with +x indicating on for x amount of time and –x indicating off for x amount of time.  That means the lead in is +2400 -600.

 

After the lead-in, the first 7 bytes are the command code, least significant bit first.  This allows for 128 command codes.  This means that 1010001 as the first 7 bits equals 69.  If you reverse the order and put it in a calculator that does binary, you will get the right decimal equivalent.  To send those bits, +600 -600 is sent if the bit is a 0, and +1200 -600 is sent if it is a 1.  That means 69 would be sent as:

 

+1200 -600 +600 -600 +1200 -600 +600 -600 +600 -600 +600 -600 +1200 -600

|      1       |      0      |        1     |      0      |       0     |     0      |       1      |   

       1                            + 4                                                         + 64     = 69

 

Then the device code is sent.  It can be 5 bits, 8 bits or 13 bits.  The lowest number of bits possible is always used.  That means device codes 0-31 use a 5 bit device code, codes 32-255 use an 8 bit device code, and codes higher than 256 would use a 13 bit device code.  However, most of the 13 bit device codes use the same value for the first 5 of the 13 bits, usually 26, which is 01011 with the least significant bit first.  Since they do that, the 13 bit Sony codes are listed with the first bits in front of a decimal point or period, which acts as a delimiter, and the other 8 bits after the dot.  If you ever find Sony codes listed elsewhere with a single number higher than 255, you can use this formula to calculate the form used here:

 

(x Mod 32) . ((x – (x Mod 32)) / 32)

 

The “Mod” function is on the standard Windows calculator.  If you can’t do this, you can assume that (x Mod 32) is 26, and modify the formula.  If you get a whole number for the second part, it is correct.

 

After the device code is sent, the light is turned off until a total time of 45000 us has been reached.  This results in the final -600 being combined with a variable off time at the end, which makes the last part of a Sony IR signal some big – number.  After 45000 us since the end of the signal, the signal is sent again, and will be sent as long as the button is held down.  Sony devices need to see at least 3 repeats before they will accept the signal, so if you don’t hold the button down for over 90000 us (.09 seconds), Sony and One For All remotes know to send the signal 3 times.  This is the only form of redundancy or error checking used in Sony signals.  Sony usually makes sure its remotes send out a strong signal so that the devices can read the code.
